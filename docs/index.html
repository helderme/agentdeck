Sessions

Lists your Claude Code and Codex sessions with title, folder and "where we left off". Resume in one click, open in VS Code (folder + the conversation in the extension), start a new session choosing agent, model, effort and mode. Favorite, pin, archive, rename, export/import.

Workspaces

Group several repos. Open them all in a multi-root VS Code window, or start a single session that sees all of them (root + the others via --add-dir) — multi-root in the terminal.

Processes

What's still running in the background — dev servers, containers — with PID, uptime, port and folder. Stop it (by PID, port or container), restart it, view the log, or start a new process.

Get started in ~2 minutes

You only need Bun. It's a Linux tool.

# 1) install Bun (once)
curl -fsSL https://bun.sh/install | bash

# 2) clone and run
git clone https://github.com/helderme/agentdeck.git
cd agentdeck
./start.sh

The start.sh script boots the server and opens a clean app window. Without it: bun server.ts and open http://localhost:7799. Port taken? AGENTDECK_PORT=8080 ./start.sh. For a menu shortcut, run ./install.sh.

Secure by default

The panel runs local commands, so it's designed to never leave your machine.

  • Only on 127.0.0.1 — nothing exposed; don't port-forward it.
  • Anti-rebinding & anti-CSRF — validates the Host header and the origin of every POST (fail-closed); a web page open in your browser triggers nothing.
  • No shell where it matters — external commands run via execFile or with escaped quotes; kill only acts on recognized processes.
  • Never exposes the token — from your login it reads only email/plan, never the credential.
