Sessões

Lista as sessões do Claude Code e do Codex com título, pasta e "onde paramos". Retoma num clique, abre no VS Code (pasta + a conversa na extensão), cria sessão nova escolhendo agente, modelo, effort e modo. Favorita, fixa, arquiva, renomeia, exporta/importa.

Workspaces

Agrupe vários repos. Abra todos numa janela multi-root do VS Code, ou inicie uma sessão só que enxerga todos eles (raiz + os outros via --add-dir) — o multi-root no terminal.

Processos

O que ficou rodando em background — dev servers, containers — com PID, tempo, porta e pasta. Finalize (por PID, porta ou container), reinicie, veja o log ou inicie um processo novo.

Começar em ~2 minutos

Só precisa do Bun. É uma ferramenta de Linux.

# 1) instale o Bun (uma vez só)
curl -fsSL https://bun.sh/install | bash

# 2) clone e rode
git clone https://github.com/helderme/agentdeck.git
cd agentdeck
./start.sh

O start.sh sobe o servidor e abre numa janela de app limpa. Sem ele: bun server.ts e abra http://localhost:7799. Porta ocupada? AGENTDECK_PORT=8080 ./start.sh. Pra um atalho no menu de aplicativos, rode ./install.sh.

O que precisa instalado

Seguro por padrão

O painel executa comandos locais, então é desenhado pra nunca sair da sua máquina.

  • Só em 127.0.0.1 — nada exposto pra fora; não encaminhe a porta.
  • Anti-rebinding e anti-CSRF — valida o header Host e a origem de todo POST (fail-closed); uma página web aberta no navegador não aciona nada.
  • Sem shell onde importa — comandos externos rodam por execFile ou com aspas escapadas; kill só atua em processos reconhecidos.
  • Nunca expõe o token — do login só lê e-mail/plano, jamais a credencial.
